# 🌴 worktree

**worktree**는 GDGoC KU에서 진행하는 모든 프로젝트를 통칭합니다. 멤버들이 실전 개발 경험을 쌓고, 팀 협업 역량을 키울 수 있는 핵심 활동입니다.

## 📌 worktree란?

`git worktree`는 하나의 레포지토리에서 여러 작업 디렉터리를 생성하여 병렬로 작업할 수 있게 하는 커맨드입니다. GDGoC KU에서는 SWE와 Designer가 여러 프로젝트에 동시에 참여할 수 있으며, 각자의 관심사와 목표에 맞는 프로젝트를 선택할 수 있습니다.

## 🎯 worktree의 목적

- **실전 개발 경험 축적**: 기획부터 배포까지 전체 개발 프로세스 경험
- **협업 역량 강화**: 개발자, 디자이너와의 실질적인 팀 프로젝트 수행
- **포트폴리오 구축**: 완성도 있는 프로젝트로 개인 포트폴리오 강화
- **기술 스택 확장**: 새로운 기술과 도구를 프로젝트에 직접 적용

## 📋 프로젝트 진행 방식

### 1. 프로젝트 제안

- 프로젝트 아이디어 제안서 작성
- 목표, 기술 스택, 예상 일정, 필요 인원 명시
- 학술팀에 제출

### 2. 팀 구성

- 관심 있는 멤버들이 자유롭게 지원
- PM 또는 팀장이 지원자 중 선정
- 최소 3명 이상 권장

### 3. 개발 진행

- **주 1회 미팅**: 진행 상황 공유, 이슈 논의
- **코드 리뷰**: 멤버 간 코드 리뷰 문화 형성

### 4. 프로젝트 마무리

- **최종 발표**: 연대와의 연합 Demoday에서 프로젝트 결과 발표
- **배포**: 가능한 프로젝트는 실제 서비스 배포


## 💡 프로젝트 예시

### 커뮤니티 도구

- 출석 체크 시스템
- 활동 포인트 관리 플랫폼
- fetch 발표 자료 아카이브

### 외부 서비스

- 캠퍼스 맛집 추천 앱
- 스터디 매칭 플랫폼
- AI 기반 학습 도우미

### 오픈소스

- 라이브러리 개발
- 도구 및 유틸리티 제작
- 기술 문서 번역 프로젝트
